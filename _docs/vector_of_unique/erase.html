Erases the specified elements from the container.

1) Removes the element at pos.
2) Removes the elements in the range [first, last).

Iterators (including the end() iterator) and references to the elements at or after the point of the erase are invalidated.

The iterator pos must be valid and dereferenceable. Thus the cend() iterator (which is valid, but is not dereferenceable) cannot be used as a value for pos.

The iterator first does not need to be dereferenceable if first == last: erasing an empty range is a no-op.

Parameters

pos - iterator to the element to remove
first, last - range of elements to remove
Type requirements
-
T must meet the requirements of MoveAssignable.

Return value

Iterator following the last removed element.

1) If pos refers to the last element, then the cend() iterator is returned.
2) If last == end() prior to removal, then the updated cend() iterator is returned.
If [first, last) is an empty range, then last is returned.

Complexity

Given an instance c of vector_of_unique:

Erase elements in unordered_set Erse elements in vector
1)
  Average case: constant.
  Worst case: c.size().
Linear: the number of calls to the destructor of T is the same as the number of elements erased, the number of calls to the assignment operator of T is no more than the lesser of the number of elements before the erased elements and the number of elements after the erased elements.
2)
Average case: std::distance(first, last).
Worst case: c.size().

Exceptions

Does not throw unless an exception is thrown by the assignment operator of T.
